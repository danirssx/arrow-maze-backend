## Team Modifications Pending Human Review

- Infrastructure layer change (LevelMapper) and application layer change (GetLevelUseCase, DailyChallengeTypes) require human review.
- `z` is optional in DB records (`z?: number`) to maintain backward-compat with pre-B7 rows that store only `{row, col}`. Reading uses `?? 0` which is safe because `Position.create` already defaults `z = 0`.
- `DailyChallengeTypes.ts` type update is a contract change — any Gemini-generated daily challenge stored in the cache before B7 will lack `z` in its JSONB. The loose `isLevelRecord` guard (`typeof value === 'object' && value !== null`) means it will still deserialize; the client will receive `z: undefined` for those older records. This is acceptable for academic scope but should be noted for production hardening.

## Lessons / Limitations

- `Position.create` has `z = 0` as default parameter, which means `Position.create(row, col, undefined)` already returns z=0 without needing `?? 0`. The explicit `?? 0` was added for clarity and is semantically correct.
- Refactoring `isArrowRecord` and `isBoardShapeRecord` to share `isPositionCell` simplified the z-optional check but surfaced pre-existing mutation debt in those guards (null checks, per-field type checks). These were not killed before B7 either.
- OpenAPI: `direction` enum was missing `FORWARD` and `BACK` — added as part of B7 since they're needed for 3D arrows that appear in the response payload.
